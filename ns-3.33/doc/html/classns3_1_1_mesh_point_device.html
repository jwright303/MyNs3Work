Detailed Description

Virtual net device modeling mesh point.

Mesh point is a virtual net device which is responsible for

  • Aggregating and coordinating 1..* real devices – mesh interfaces, see MeshInterfaceDevice class.
  • Hosting all mesh-related level 2 protocols.

One of hosted L2 protocols must inplement L2RoutingProtocol interface and is used for packets forwarding.

From the level 3 point of view MeshPointDevice is similar to BridgeNetDevice, but the packets, which going through may be changed (because L2 protocols may require their own headers or tags).

Config Paths

ns3::MeshPointDevice is accessible through the following paths with Config::Set and Config::Connect:

  • "/NodeList/[i]/DeviceList/[i]/$ns3::MeshPointDevice"

Attributes

  • Mtu: The MAC-level Maximum Transmission Unit
    • Set with class: ns3::UintegerValue
    • Underlying type: uint16_t 0:65535
    • Initial value: 65535
    • Flags: construct write read
  • RoutingProtocol: The mesh routing protocol used by this mesh point.
    • Set with class: ns3::PointerValue
    • Underlying type: ns3::Ptr< ns3::MeshL2RoutingProtocol>
    • Initial value: 0x0
    • Flags: construct write read

No TraceSources are defined for this type.
Size of this type is 160 bytes (on a 64-bit architecture).
